# AI Sync Manager 产品测试流程

## 测试环境准备

### 1.1 环境要求

- **操作系统**：Windows 10+、Linux、macOS
- **Go 版本**：1.21 或更高
- **终端**：支持 PowerShell（Windows）、Bash（Linux/macOS）
- **AI 工具**：至少安装一个（Codex 或 Claude）

### 1.2 前置检查清单

- [ ] 已克隆项目代码库
- [ ] 已检查 Go 版本（`go version`）
- [ ] 确认终端可用
- [ ] 确认已安装至少一个 AI 工具

### 1.3 构建程序

```bash
# 克隆项目
git clone <repository-url>
cd ai-sync-manager

# 构建程序
make build

# 验证构建成功
ls -la bin/
```

**预期结果**：
- `bin/` 目录存在
- 存在 `ai-sync.exe`（Windows）或 `ai-sync`（Linux/macOS）

## 测试用例 1：扫描配置功能

### 测试目标

验证 `scan` 命令能够正确扫描本机 AI 工具配置。

### 测试步骤

#### 步骤 1：扫描所有工具

**执行命令**：

```bash
./bin/ai-sync scan
```

**预期结果**：
- 显示 Codex 检测结果（如果已安装）
- 显示 Claude 检测结果（如果已安装）
- 显示配置目录路径
- 显示可同步项数量
- 显示关键配置和扩展内容

**验证点**：
- [ ] 输出包含"检测结果"
- [ ] 输出包含"配置目录"
- [ ] 输出包含"可同步项"
- [ ] 如果工具已安装，状态为"可同步"
- [ ] 如果工具未安装，状态为"不可同步"

---

#### 步骤 2：扫描特定工具

**执行命令**：

```bash
./bin/ai-sync scan codex
./bin/ai-sync scan claude
```

**预期结果**：
- 只显示指定工具的扫描结果
- 不显示其他工具信息

**验证点**：
- [ ] `scan codex` 只显示 Codex 相关信息
- [ ] `scan claude` 只显示 Claude 相关信息

---

#### 步骤 3：查看扫描规则

**执行命令**：

```bash
./bin/ai-sync scan rules
```

**预期结果**：
- 显示默认全局规则
- 显示自定义规则（如果有）
- 显示已注册项目（如果有）

**验证点**：
- [ ] 包含"默认全局规则"部分
- [ ] 每条规则显示文件名、路径、作用域

---

#### 步骤 4：添加自定义规则

**执行命令**：

```bash
./bin/ai-sync scan add codex <绝对路径>/config.toml
```

**预期结果**：
- 规则添加成功
- 重新扫描后能识别到该文件

**验证点**：
- [ ] 添加成功无错误
- [ ] `scan rules` 显示新增的自定义规则

---

### 测试场景

| 场景 | 命令 | 预期结果 |
|------|------|----------|
| 已安装 Codex | `scan codex` | 状态为"可同步" |
| 未安装 Codex | `scan codex` | 状态为"不可同步" |
| 添加自定义规则 | `scan add` | 规则列表包含新规则 |
| 删除规则 | `scan remove` | 规则列表不包含已删除规则 |
