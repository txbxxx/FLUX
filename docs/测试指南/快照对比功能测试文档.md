# 快照对比功能测试文档

> **功能名称**：snapshot diff — 快照对比
> **版本**：v0.3
> **编写日期**：2026-04-11
> **测试环境**：Windows 11, Go 1.25+, fl CLI

---

## 1. 功能概述

`fl snapshot diff` 命令支持两种对比场景：

- **快照 vs 快照**：对比两个快照之间的配置差异
- **快照 vs 文件系统**：对比快照与当前磁盘上的配置差异

分级输出：默认摘要模式、`-v` 内容级 diff、`--side-by-side` 并排显示。

---

## 2. 前置条件

1. 已构建 `fl` 二进制（`make build`）
2. 至少存在一个快照（通过 `snapshot create` 创建）
3. 快照对应的工具配置目录可访问

## 3. 测试用例

### 3.1 命令帮助

| 编号 | 测试场景 | 操作 | 预期结果 |
|------|----------|------|----------|
| T01 | 查看帮助 | `fl snapshot diff --help` | 显示命令用法、参数说明、Flags 列表 |
| T02 | 无参数调用 | `fl snapshot diff` | 报错：需要至少 1 个参数 |
| T03 | 超过 2 个参数 | `fl snapshot diff a b c` | 报错：最多 2 个参数 |

### 3.2 快照 vs 文件系统（单参数）

| 编号 | 测试场景 | 操作 | 预期结果 |
|------|----------|------|----------|
| T10 | 有差异 - 摘要模式 | `fl snapshot diff <id>` | 显示变更文件列表 + 统计行；退出码 1 |
| T11 | 有差异 - 内容 diff | `fl snapshot diff <id> -v` | 显示 unified diff 格式（`@@` 块、`+`/`-` 行）；退出码 1 |
| T12 | 无差异 | `fl snapshot diff <id> <id>`（两个相同快照） | 输出"无差异"；退出码 0 |
| T13 | 按名称引用 | `fl snapshot diff "备份名称"` | 正确解析名称并显示差异 |
| T14 | 快照不存在 | `fl snapshot diff "不存在的名称"` | 报错提示未找到快照 |

### 3.3 快照 vs 快照（双参数）

| 编号 | 测试场景 | 操作 | 预期结果 |
|------|----------|------|----------|
| T20 | 两个不同快照 | `fl snapshot diff <id1> <id2>` | 显示两快照间的差异 |
| T21 | 两个相同快照 | `fl snapshot diff <id> <id>` | 输出"无差异"；退出码 0 |
| T22 | 源快照不存在 | `fl snapshot diff "不存在" <id>` | 报错提示源快照未找到 |
| T23 | 目标快照不存在 | `fl snapshot diff <id> "不存在"` | 报错提示目标快照未找到 |

### 3.4 过滤功能

| 编号 | 测试场景 | 操作 | 预期结果 |
|------|----------|------|----------|
| T30 | 工具类型过滤 | `fl snapshot diff <id> --tool claude` | 只显示 claude 工具的变更文件 |
| T31 | 工具类型过滤 - 无匹配 | `fl snapshot diff <id> --tool nonexistent` | 无差异输出（过滤后为空） |
| T32 | 路径模式过滤 | `fl snapshot diff <id> --path "settings*"` | 只显示路径匹配的文件 |
| T33 | 路径模式过滤 - 无匹配 | `fl snapshot diff <id> --path "nonexistent/*"` | 无差异输出 |
| T34 | 工具 + 路径组合 | `fl snapshot diff <id> --tool claude --path "settings*"` | 同时满足两个条件 |

### 3.5 输出模式

| 编号 | 测试场景 | 操作 | 预期结果 |
|------|----------|------|----------|
| T40 | 摘要模式（默认） | `fl snapshot diff <id>` | `git diff --stat` 风格：文件名 + 状态 + 行数统计 |
| T41 | 内容 diff | `fl snapshot diff <id> -v` | unified diff 格式，含 `@@` 块头、`+`/`-` 行着色 |
| T42 | 并排显示 | `fl snapshot diff <id> -v --side-by-side` | 左右分栏，`\|` 分隔符，差异行着色 |
| T43 | 颜色控制 - never | `fl snapshot diff <id> --color never` | 无 ANSI 颜色码 |
| T44 | 颜色控制 - always | `fl snapshot diff <id> --color always` | 强制 ANSI 颜色码 |

### 3.6 二进制文件

| 编号 | 测试场景 | 操作 | 预期结果 |
|------|----------|------|----------|
| T50 | 含二进制文件 | 对比包含二进制文件（如图片）的快照 | 显示 `Binary files differ` + 大小变化 |

## 6. 已知限制

1. `--side-by-side` 在终端宽度不足 80 列时提示切换为 `-v`
2. `--path` 使用 `filepath.Match`，仅支持 glob 模式（如 `*.json`、`mcp/*`），不支持正则
3. 二进制文件不显示行级差异，仅标记 `Binary files differ`
