### 2. 核心功能

#### 加密算法
- **AES-256-GCM**: 高级加密标准，GCM 认证模式
- **Nonce**: 每次加密生成随机 nonce，确保相同明文产生不同密文
- **Base64 编码**: 加密结果使用 Base64 编码存储

#### 密钥管理
密钥获取优先级：
1. **环境变量**: 从 `KeyEnvVar` 指定的环境变量读取
2. **密钥文件**: 从 `KeyPath` 指定的文件读取
3. **自动生成**: 自动生成 32 字节随机密钥并保存到 `~/.ai-sync-manager/.encryption_key`

#### 加密服务方法

```go
type Service struct {
    config *models.EncryptionConfig
    key    []byte
}

// 核心方法
Encrypt(plaintext string) (*EncryptionResult, error)

// 便捷方法

DecryptPassword(encrypted string) (string, error)
EncryptToken(token string) (string, error)
DecryptToken(encrypted string) (string, error)
EncryptSSHKey(sshKey string) (string, error)
DecryptSSHKey(encrypted string) (string, error)

// 敏感数据方法
EncryptSensitiveData(dataType SensitiveType, value string) (*SensitiveData, error)
DecryptSensitiveData(data *SensitiveData) (string, error)
```

## 技术亮点

### 1. 安全性
- **AES-256-GCM**: 提供认证加密，防止数据篡改
- **随机 Nonce**: 每次加密使用不同 nonce，防止模式攻击
- **密钥分离**: 密钥与数据分离存储，支持多种密钥来源

### 2. 灵活性
- **可配置**: 支持启用/禁用加密
- **多密钥源**: 支持环境变量、文件、自动生成
- **便捷方法**: 提供针对不同类型数据的便捷加解密方法

### 3. 错误处理
- **优雅降级**: 解密失败时返回错误信息而非 panic
- **清晰日志**: 记录加密操作的关键步骤

```
敏感数据 (Password/Token/SSHKey)
    ↓

    ↓
Crypto Service (Encrypt)

    ↓
```

## 使用示例

### 配置加密服务
```go
// 配置加密（使用环境变量）
config := &models.EncryptionConfig{
    Enabled:   true,
    Algorithm: "aes256-gcm",
    KeyEnvVar: "MY_ENCRYPTION_KEY",
}
cryptoSvc, _ := crypto.NewService(config)

// 加密密码
encrypted, _ := cryptoSvc.EncryptPassword("my-password")

// 解密密码
decrypted, _ := cryptoSvc.DecryptPassword(encrypted)
```

### App 层使用
```typescript
// 前端调用
const status = await GetEncryptionStatus()
console.log(status.enabled, status.algorithm)

const encrypted = await EncryptPassword("my-password")
console.log(encrypted)

const decrypted = await DecryptPassword(encrypted)
console.log(decrypted) // "my-password"
```

## 安全考虑

1. **密钥保护**
   - 密钥文件权限设置为 0600（仅所有者可读写）
   - 环境变量优先，避免密钥泄露到版本控制

2. **数据安全**
   - 使用 GCM 模式，提供认证加密
   - 加密失败不会泄露原始数据

3. **默认安全**
   - 默认不启用加密，需要用户主动配置
   - 自动生成的密钥存储在用户目录
