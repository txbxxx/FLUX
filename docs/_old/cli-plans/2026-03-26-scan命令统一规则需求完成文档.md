## 4. 当前规则行为说明

### 4.1 默认全局规则

Codex 默认全局规则：

- `commands/`
- `skills/`
- `output-styles/`
- `plugins/blocklist.json`
- `plugins/installed_plugins.json`
- `plugins/known_marketplaces.json`

### 4.2 项目规则模板

Codex 项目规则模板：

- `.codex/`
- `AGENTS.md`

Claude 项目规则模板：

- `.claude/`
- `CLAUDE.md`

说明：

- 项目规则模板不会默认生效
- 只有在用户通过 `scan add --project ...` 注册项目后才参与最终规则解析

### 4.3 Claude `plugins/` 默认同步范围

当前默认只同步以下 3 个文件：

- `plugins/blocklist.json`
- `plugins/installed_plugins.json`
- `plugins/known_marketplaces.json`

本策略依据是“最小可恢复状态”原则，而不是“官方强制要求”。

### 4.4 关键文件门槛

当前关键文件门槛如下：

- Codex：`~/.codex/config.toml`
- Claude：`~/.claude/settings.json`

规则说明：

- 找到配置目录但缺少关键文件时，状态为“暂不可同步”
- 只有关键文件满足且最终规则下存在命中项时，状态才会进入“可同步”

## 5. 当前边界与约束

以下边界是当前实现的设计结果，不属于缺陷：

1. 默认规则只覆盖全局配置，不自动枚举项目目录。
2. 绝对路径支持仅用于已登记规则，不放开成任意本机路径访问。
3. `scan add <app> <absolute-file-path>` 当前只支持文件，不支持目录。
4. `scan add --project ...` 当前只登记项目根目录，不支持自定义项目规则模板。
5. `snapshot create --project-path` 当前保留兼容参数；统一规则模式下，项目来源以已注册项目为准。

## 6. 手工验收清单

以下命令可作为交付后的手工验收清单：

### 6.1 扫描验收

```powershell
fl scan
fl scan codex
fl scan claude
fl scan codex claude
fl scan list
fl scan list claude
```

验收点：

- 无参数时返回全部支持工具
- 指定 app 时按传入顺序输出
- 输出包含中文检测结果、配置目录、可同步项数量、原因
- `scan` 与 `scan list` 都用于查看当前扫描结果
- `scan rules` 用于查看默认规则、项目模板、已注册项目和自定义规则

### 6.2 规则管理验收

```powershell
fl scan add claude C:\Users\<user>\.claude.json
fl scan add --project codex demo D:\workspace\demo
fl scan list
fl scan remove claude C:\Users\<user>\.claude.json
fl scan remove --project codex D:\workspace\demo
```

验收点：

- 绝对路径文件规则可新增、列出、删除
- 项目可注册、列出、删除
- 重复添加时不会产生脏数据

### 6.3 `get` 验收

```powershell
fl get claude settings.json
fl get claude C:\Users\<user>\.claude.json
fl get codex D:\workspace\demo\.codex\config.toml
```

验收点：

- 默认规则路径可读取
- 已登记绝对路径可读取
- 已注册项目扫描对象下的已识别路径可读取
- 未登记越界路径应被拒绝

### 6.4 快照验收

```powershell
fl snapshot create --tools claude --message "verify unified rules"
fl snapshot create --tools codex,claude --message "verify unified rules both" --scope both
```

验收点：

- 仅最终规则命中的文件进入快照
- Claude `plugins/` 非白名单文件不会进入快照
- 已登记绝对路径文件会进入快照
- 已注册项目扫描对象下的已识别路径会进入快照
