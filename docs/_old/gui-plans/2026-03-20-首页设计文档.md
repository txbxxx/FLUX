# AI Sync Manager 首页设计文档

## 文档信息

- 日期：2026-03-20
- 范围：首页
- 当前状态：已确认方向与布局结构
- 设计目标：将首页设计为专业工具型的快速启动台，而不是传统欢迎页或重信息仪表盘

## 1. 设计结论

首页采用“专业工具型的快速开始页”方案。

核心判断如下：

- 首页第一职责是帮助用户立即开始关键操作
- 首页不是纯展示型欢迎页
- 首页不是以数据监控为核心的仪表盘
- 页面应优先突出主流程入口，而不是承载过多历史信息

因此，首页定位为：

> 一个面向桌面端用户的任务启动台，用最短路径把用户带入扫描、快照、仓库连接和恢复流程。

## 2. 方案选择过程

讨论过的备选方案有三种：

### 方案 A：任务启动台

首页主体由若干高优先级操作卡组成，用户打开后直接进入关键动作。

优点：

- 上手快
- 信息层级明确
- 非常适合第一次使用和高频回访

缺点：

- 历史信息承载能力相对较弱

### 方案 B：双栏控制台

左侧为快速操作，右侧为状态和最近记录。

优点：

- 同时兼顾操作和状态
- 更接近成熟工具后台

缺点：

- 首屏行动感会被状态信息分散

### 方案 C：步骤式欢迎页

按照扫描、选择、生成、同步等步骤做成强引导流程。

优点：

- 对新用户非常友好

缺点：

- 回访用户效率较差
- 更像 onboarding，而不是长期使用的首页

最终采用：

- 以方案 A 为主
- 吸收方案 B 的少量状态摘要能力

## 3. 首页定位

首页是“快速开始操作”的入口页，其职责有三项：

1. 告诉用户当前环境是否已准备好
2. 告诉用户下一步最值得做什么
3. 提供最短路径进入核心流程

首页不承担以下职责：

- 不做完整历史管理页
- 不做完整同步详情页
- 不做复杂设置承载页
- 不做纯品牌展示页

## 4. 页面结构

首页采用三段式结构，面向桌面端优先设计。

### 4.1 顶部：环境状态横幅

这一层用于快速确认“当前是否可以开始同步操作”。

建议包含以下内容：

- 产品名：AI Sync Manager
- 一句简短说明：用于解释这是 AI 配置同步工具，而不是通用网盘
- 工具检测状态
- 仓库连接状态
- 加密状态

信息表达方式：

- 使用横向状态胶囊或标签
- 每个状态使用明确文案
- 避免使用过长说明文字

这一块的目标不是介绍产品，而是让用户在 3 秒内完成环境判断。

### 4.2 中部：主操作区

这是首页的核心区域，视觉权重最高。

布局建议：

- 使用 2 x 2 网格
- 四张大卡片组成主要操作入口
- 每张卡片可整卡点击
- hover 反馈仅通过边框、底色、阴影变化完成
- 不使用缩放动效，保持专业工具感

四张卡片为：

#### 扫描工具

用于检测本机 AI 工具安装与配置情况。

卡片内容建议：

- 标题：扫描工具
- 说明：检测本机可同步的 AI 工具与配置目录
- 主按钮：立即扫描
- 辅助状态：例如“已检测到 2 个工具”或“尚未扫描”

#### 创建快照

用于生成当前配置状态的本地快照。

卡片内容建议：

- 标题：创建快照
- 说明：保存当前配置，便于同步、回滚和迁移
- 主按钮：创建快照
- 辅助状态：例如“最近快照：2 小时前”

#### 连接仓库

用于建立 Git 仓库同步通道。

卡片内容建议：

- 标题：连接仓库
- 说明：配置远程仓库地址与认证方式
- 主按钮：连接仓库
- 辅助状态：例如“未连接”或“已连接 main 分支”

#### 拉取并恢复

用于从远端同步配置并恢复到本机。

卡片内容建议：

- 标题：拉取并恢复
- 说明：拉取远端快照并应用到当前设备
- 主按钮：开始恢复
- 辅助状态：例如“可恢复最近快照”或“尚无远端记录”

### 4.3 底部：近期记录区

这一层用于承接最近工作，不在首页展示完整表格。

建议采用左右两栏：

- 左栏：最近快照
- 右栏：最近同步或恢复记录

每栏建议只展示 3 条记录，包含：

- 名称
- 时间
- 简短结果状态

设计原则：

- 只展示摘要
- 支持点击进入详情页
- 不在首页承担复杂筛选和管理功能

## 5. 设计原则

首页视觉和交互遵守以下原则：

- 优先操作，而不是优先展示
- 优先桌面工具效率，而不是营销感包装
- 保持结构克制、信息密度可读
- 首页元素必须服务于“下一步操作”
- 状态信息只做判断辅助，不喧宾夺主

## 6. 当前已确认范围

目前已确认的内容包括：

- 首页优先承担“快速开始操作”职责
- 整体风格采用专业工具型
- 结构采用三段式布局
- 主体采用四张操作卡作为一级入口
- 底部保留近期记录摘要区

## 7. 视觉系统建议

首页视觉基调采用“冷静、可信、偏工程工具”的路线。

目标不是做营销型首页，而是建立一种可靠、稳健、专业的桌面工具气质。

### 7.1 色彩方向

建议采用以下主色逻辑：

- 主背景：深石墨色
- 卡片背景：比主背景亮一层的深灰蓝
- 主强调色：低饱和蓝青色
- 成功色：稳重绿色
- 警告色：琥珀色
- 错误色：偏暗红色

建议的视觉判断标准：

- 不使用纯黑背景
- 不使用高饱和紫色作为主品牌色
- 不使用消费级 SaaS 常见的大面积明亮渐变
- 重点依靠层级和对比而不是夸张颜色制造视觉焦点

这种色彩系统适合：

- Git / 同步 / 快照 / 配置管理类工具
- 长时间桌面停留使用
- 与技术信息密度较高的界面结合

### 7.2 字体方向

当前默认的圆润字体不适合作为专业工具型首页的主字体。

建议的字体策略：

- 正文：以高可读性无衬线字体为主
- 标题：使用更紧凑、更利落的无衬线风格
- 技术信息：路径、分支名、状态码、时间戳统一使用等宽字体

这样可以形成更清晰的信息气质区分：

- 正文负责阅读舒适度
- 标题负责建立工具感和模块边界
- 等宽文本负责承接技术信息

### 7.3 图标与图形语言

建议使用统一的线性图标系统。

要求如下：

- 不使用 emoji
- 不混用不同风格图标
- 首页主操作卡图标需一眼区分含义
- 图标大小统一，保持稳定视觉节奏

推荐的图形语言：

- 扫描工具：雷达、搜索、网格扫描
- 创建快照：相机、层叠、存档
- 连接仓库：分支、仓库、链接
- 拉取并恢复：下载、回退、同步箭头

### 7.4 卡片风格

首页卡片应体现“可操作的工具模块”，而不是营销卡片。

卡片风格建议：

- 中等圆角
- 阴影克制
- 边框清晰
- 背景层次明显但不过度透明
- 依靠标题、说明、状态、按钮四层建立内容秩序

hover 状态建议：

- 边框提亮
- 背景略微上浮一层
- 图标或主按钮颜色增强

不建议：

- 卡片缩放
- 大位移动画
- 过强玻璃拟态

## 8. 首页组件清单

首页建议由以下组件组成：

### 8.1 AppHeader

作用：

- 显示产品名称
- 显示当前页面定位
- 提供全局入口（后续可扩展设置、帮助、窗口控制区）

建议内容：

- Logo / 产品名
- 一句短说明
- 右侧保留轻量操作位

### 8.2 EnvironmentStatusBar

作用：

- 汇总展示当前系统是否具备执行主流程的前提条件

建议状态项：

- 工具检测状态
- 仓库连接状态
- 加密状态

状态样式建议：

- 每项使用独立状态胶囊
- 采用图标 + 文案组合
- 文案应明确，如“已检测”“未连接”“已启用”

### 8.3 QuickActionGrid

作用：

- 承载四张一级操作卡

布局建议：

- 桌面端 2 x 2
- 中等间距
- 卡片尺寸一致

### 8.4 QuickActionCard

每张卡片统一结构如下：

- 图标
- 标题
- 一句说明
- 当前状态
- 主按钮
- 次级提示文本（可选）

这类卡片应支持整卡点击，但主按钮仍需保留明确视觉锚点。

### 8.5 RecentSnapshotList

作用：

- 展示最近快照记录摘要

每条记录建议包含：

- 快照名
- 创建时间
- 涉及工具数量或范围
- 状态标签

### 8.6 RecentSyncList

作用：

- 展示最近同步、拉取、恢复等操作摘要

每条记录建议包含：

- 操作类型
- 时间
- 结果状态
- 简短描述

## 9. 状态设计

首页需要重点处理以下三类状态：

### 9.1 空状态

常见场景：

- 尚未扫描工具
- 尚未创建任何快照
- 尚未连接远程仓库
- 尚无同步记录

空状态设计原则：

- 不做空白区域
- 必须给出下一步可执行动作
- 文案短而明确，不写成长说明

示例方向：

- “尚未扫描本机工具，先完成检测再创建快照”
- “还没有任何快照，建议先保存当前环境”
- “未连接仓库，连接后即可推送和恢复配置”

### 9.2 错误状态

首页只承载摘要级错误，不在首页展开复杂故障排查。

适合在首页展示的错误有：

- 工具扫描失败
- 仓库认证失败
- 最近一次同步失败

展示方式建议：

- 在对应卡片或状态条中显示简短错误摘要
- 提供“查看详情”或“重试”入口
- 错误文案聚焦可执行信息，而不是堆栈

### 9.3 加载状态

首页至少要覆盖以下加载场景：

- 初次进入加载环境状态
- 扫描工具执行中
- 拉取最近记录中

加载设计建议：

- 顶部状态条可用骨架或占位胶囊
- 主卡片按钮进入 loading
- 近期记录列表使用 2 到 3 行骨架屏

## 10. 首次使用与回访用户

首页需要兼顾首次使用用户和高频回访用户，但不应做成复杂双模式页面。

建议方式：

- 通过状态自动改变提示文案
- 不切换整体布局

首次使用用户的首页特点：

- 状态区大概率为“未扫描”“未连接”“无快照”
- 主操作卡应更强调引导性文案
- 默认推荐第一步为“扫描工具”

回访用户的首页特点：

- 状态区展示最近结果
- 主操作卡更强调效率和最近状态
- 底部近期记录更有价值

## 11. 与现有 API 的首页映射

首页设计应尽量只依赖当前已经明确可用的 API 能力。

建议的首页映射关系如下：

- 工具检测状态 / 扫描工具
  - `ScanTools`
  - `ScanToolsWithProjects`

- 快照摘要 / 创建快照
  - `ListSnapshots`
  - `CreateSnapshot`

- 环境状态
  - `GetVersion`
  - `GetSystemInfo`
  - `HealthCheck`
  - `GetEncryptionStatus`

- 同步记录摘要
  - 现阶段可先使用已有本地快照和操作结果占位
  - 对未完成接口保持预留态，不在首页承诺完整远端能力

注意：

- 文档中列出的部分接口尚未实现
- 首页应明确区分“可立即执行”和“后续扩展能力”
- 不建议在首页直接依赖尚未落地的远端快照完整能力

### 13.2 布局要点

- 页面宽度按桌面应用主窗口设计
- 主内容建议控制在稳定最大宽度内，避免超宽导致卡片分散
- 顶部状态区与中部操作区之间保留明确留白，形成任务层次
- 最近记录区权重低于操作区，但仍保持可读性

### 13.3 栅格建议

建议采用 12 列栅格思路，但实现时不必强依赖传统网页栅格框架。

建议尺寸逻辑：

- 页面内容容器：约 1200 到 1320px 上限
- 主操作区：双列平均分布
- 两列间距：24px 左右
- 模块间垂直间距：24 到 32px
- 卡片内边距：24px

## 14. 首页文案建议

首页文案应保持简短、直接、可执行，避免品牌宣传式语言。

### 14.1 页面标题区

主标题建议：

- 快速开始

副标题建议：

- 从扫描本机工具开始，保存快照并同步到你的配置仓库。

可选替代文案：

- 管理你的 AI coding 环境，从一次扫描开始。
- 扫描、保存、同步、恢复，保持多设备配置一致。

### 14.2 状态条文案

工具状态：

- 未扫描
- 已检测 2 个工具
- 检测失败

仓库状态：

- 未连接仓库
- 已连接到 main
- 认证失败

加密状态：

- 未启用加密
- 已启用加密
- 密钥缺失

### 14.3 主操作卡文案

扫描工具卡：

- 标题：扫描工具
- 描述：检测本机可同步的 AI 工具与配置目录
- 主按钮：立即扫描
- 状态提示：尚未执行检测

创建快照卡：

- 标题：创建快照
- 描述：保存当前环境，便于同步、回滚和迁移
- 主按钮：创建快照
- 状态提示：当前没有可用快照

连接仓库卡：

- 标题：连接仓库
- 描述：配置远程仓库地址与认证方式
- 主按钮：连接仓库
- 状态提示：尚未配置远程同步源

拉取并恢复卡：

- 标题：拉取并恢复
- 描述：从远端拉取快照并应用到当前设备
- 主按钮：开始恢复
- 状态提示：连接仓库后可恢复其他设备配置

### 14.4 空状态文案

最近快照为空：

- 还没有任何快照
- 创建第一个快照后，你可以在这里快速回到最近的环境版本

最近同步为空：

- 还没有同步记录
- 连接仓库并完成推送后，这里会显示最近操作结果

## 17. 应用导航结构决策

首页最终放入一个轻量的桌面应用壳中，而不是独立欢迎页。

采用的导航结构为：

> 左侧固定主导航 + 右侧内容工作区

### 17.1 选择理由

选择该结构的原因如下：

- 产品天然会扩展为多页面桌面工具
- 首页只是一级入口之一，不适合长期孤立存在
- 左侧导航更适合桌面端持续使用场景
- 后续扩展快照、同步、仓库、设置页面时成本最低
- 用户更容易建立稳定的空间记忆

### 17.2 导航项建议

建议的一级导航项：

- 首页
- 快照
- 同步
- 仓库
- 设置

可选补充项：

- 帮助
- 关于

### 17.3 侧栏结构

侧栏建议分为三段：

#### 顶部：品牌区

包含：

- Logo
- 产品名 `AI Sync Manager`
- 简短产品说明（可选）

#### 中部：主导航区

显示一级导航项，采用图标 + 文案结构。

交互要求：

- 当前页面高亮
- hover 提供轻量反馈
- 不做复杂展开折叠

#### 底部：辅助区

建议包含：

- 设置快捷入口
- 帮助入口
- 版本号或环境信息

### 17.4 导航风格建议

导航壳应保持“轻量”，不应做成厚重后台。

建议如下：

- 侧栏宽度适中
- 背景比主内容区略深
- 图标统一使用线性图标
- 当前项使用强调色背景或高亮边条
- 文案简洁，不超过两到三个字

不建议：

- 复杂多级菜单
- 强动画折叠
- 过多状态徽标塞入导航项

## 18. 首页高保真布局说明

以下内容描述首页在最终实现时的高保真视觉层次。

### 18.1 整体分区

```text
┌───────────────┬──────────────────────────────────────────────────────┐
│ Left Sidebar  │ Right Content Area                                   │
│               │                                                      │
│ 品牌区         │ 页面头部                                              │
│ 首页           │ 快速开始                                 搜索/帮助   │
│ 快照           │ 从扫描本机工具开始，保存快照并同步到仓库             │
│ 同步           ├──────────────────────────────────────────────────────┤
│ 仓库           │ 状态横幅                                              │
│ 设置           │ [已检测 2 项] [未连接仓库] [已启用加密]              │
│               ├──────────────────────────────────────────────────────┤
│               │ 主操作区                                              │
│               │ ┌──────────────┐ ┌──────────────┐                    │
│               │ │ 扫描工具      │ │ 创建快照      │                    │
│               │ └──────────────┘ └──────────────┘                    │
│               │ ┌──────────────┐ ┌──────────────┐                    │
│               │ │ 连接仓库      │ │ 拉取并恢复    │                    │
│               │ └──────────────┘ └──────────────┘                    │
│               ├──────────────────────────────────────────────────────┤
│               │ 近期记录区                                            │
│               │ 最近快照                     最近同步 / 恢复          │
└───────────────┴──────────────────────────────────────────────────────┘
```

### 18.2 页面头部

右侧内容区顶部保留轻量页面头部。

建议包含：

- 页面标题：快速开始
- 页面副标题：一句简洁说明
- 右上角辅助入口：帮助、搜索、全局反馈入口（按项目需要决定）

这块不要做成复杂工具栏，避免稀释首页主任务。

### 18.3 状态横幅

状态横幅应位于页面头部下方，作为“开始操作前的环境判断层”。

建议视觉表现：

- 横向整块容器
- 内部为 3 个状态模块
- 支持状态颜色区分，但以文字清晰为主

推荐信息顺序：

1. 工具检测
2. 仓库连接
3. 加密状态

### 18.4 主操作区

主操作区是整个首页的视觉焦点。

高保真层面建议：

- 四张卡片高度一致
- 图标置于卡片上方或左上角
- 标题与描述形成明确层级
- 状态提示使用次级文字
- 主按钮放在卡片底部，便于形成统一操作节奏

主操作区的目标是让用户无需思考就能点击第一步。

### 18.5 近期记录区

近期记录区权重低于主操作区，作为“回到最近工作”的辅助层。

视觉建议：

- 两个并排面板
- 每个面板顶部带标题与“查看全部”
- 列表项使用简洁行结构
- 状态标签尽量小而清晰

### 18.6 视觉层级优先级

首页的视觉权重应遵守以下顺序：

1. 主操作卡
2. 页面标题区
3. 状态横幅
4. 近期记录区
5. 侧栏辅助区

这意味着首页不是依靠大标题吸引注意，而是依靠“可行动模块”驱动用户。
