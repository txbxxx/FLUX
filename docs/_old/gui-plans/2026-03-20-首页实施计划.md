# AI Sync Manager Homepage Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Build the first production-ready frontend homepage for AI Sync Manager with a left navigation app shell, a quick-start homepage, and real homepage data wired to the existing Wails backend APIs.

**Architecture:** The frontend should move from the default Wails Vue template to a lightweight desktop application shell. Keep the first implementation narrow: add routing for top-level pages, implement the homepage as the primary view, and use a small API adapter plus a homepage-specific composable instead of introducing a large global state layer. The Wails backend already exposes the required methods in `app.go`, but the generated `frontend/wailsjs` bindings are stale, so the implementation must refresh them through the normal Wails dev/build flow before wiring real data.

**Tech Stack:** Vue 3, TypeScript, Vue Router, Vite, Vitest, @vue/test-utils, Wails 2, Go backend bindings

---

```ts
import {
  GetEncryptionStatus,
  GetVersion,
  HealthCheck,
  ListSnapshots,
  ScanTools,
} from '../../wailsjs/go/main/App'

export async function fetchHomepageData() {
  const [version, health, tools, snapshots, encryption] = await Promise.all([
    GetVersion(),
    HealthCheck(),
    ScanTools(),
    ListSnapshots(3, 0),
    GetEncryptionStatus(),
  ])

  return { version, health, tools, snapshots, encryption }
}
```

- The repository already contains unrelated local changes, so implementation should happen in an isolated worktree
