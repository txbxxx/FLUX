## 问题描述

执行 `ai-sync get` 命令时不带任何参数，错误提示为 cobra 默认的 "accepts 1 arg(s), received 0"，没有明确告知用户缺少的是 project 参数。

### 实际错误信息

```
accepts 1 arg(s), received 0
```

### 期望错误信息

应显示更友好的中文提示，例如：

```
请指定项目名称，例如：ai-sync get claude 或 ai-sync get claude settings.json
```

## 根本原因分析

### 问题代码位置

`internal/cli/cobra/get.go` 第 17-21 行：

```go
command := &spcobra.Command{
    Use:   "get <project> [path]",
    Short: "查看或编辑指定 AI 工具配置",
    Args:  spcobra.MinimumNArgs(1),
    RunE: func(cmd *spcobra.Command, args []string) error {
        // ...
    },
}
```

### 问题分析

1. 命令使用了 `spcobra.MinimumNArgs(1)` 进行参数校验
2. 当参数数量不足时，cobra 默认错误信息为英文 "accepts 1 arg(s), received 0"
3. 错误信息没有指出缺少的是 `<project>` 参数，也没有给出正确用法的提示

## 修复方案

### 推荐方案：使用自定义 Args 验证函数

在 `get.go` 中添加自定义参数验证，提供友好的中文错误提示：

```go
import (
    "fmt"
    spcobra "github.com/spf13/cobra"
)

func validateGetArgs(cmd *spcobra.Command, args []string) error {
    if len(args) == 0 {
        return fmt.Errorf("请指定项目名称，例如：ai-sync get claude 或 ai-sync get claude settings.json\n查看支持的项目：ai-sync scan list")
    }
    if len(args) > 2 {
        return fmt.Errorf("参数过多，get 命令接受 1-2 个参数：<project> [path]")
    }
    return nil
}

command := &spcobra.Command{
    Use:   "get <project> [path]",
    Short: "查看或编辑指定 AI 工具配置",
    Args:  validateGetArgs,
    // ...
}
```

## 影响范围

- `ai-sync get` 命令
- 所有首次使用该命令且不熟悉用法的用户

## 验证方法

1. 执行 `ai-sync get`（无参数），验证错误信息是否友好

3. 执行 `ai-sync get claude settings.json`，验证正常功能
