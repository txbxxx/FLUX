## 问题描述

添加远程仓库后，数据库中 `remote_configs` 表的 `id` 字段为空：

```
id    name          url                                              ...
      RemoteConfig  https://git.tanc.fun:9999/txbxxx/RemoteConfig.git ...
```

ID 为空导致后续按 ID 查找、更新、删除操作可能失败或匹配到错误记录。

## 根因分析

### 代码追踪

**UseCase 层** (`remote_method.go:113-128`)：

```go
config := &models.RemoteConfig{
    Name:      name,
    URL:       url,
    Branch:    branch,
    IsDefault: input.Project != "",
    Status:    models.StatusInactive,
    Auth:      models.AuthConfig{...},
    CreatedAt: time.Now(),
    UpdatedAt: time.Now(),
    // 注意：没有设置 ID
}
```

**DAO 层** (`remote_config.go:66-69`)：

```go
func (dao *RemoteConfigDAO) Create(config *RemoteConfig) error {
    row := remoteConfigToRow(config)
    return dao.db.GetConn().Create(&row).Error  // 直接使用 config 的 ID（空字符串）
}
```

**数据库层** (`remoteConfigRow`)：

```go
type remoteConfigRow struct {
    ID string `gorm:"column:id;primaryKey"`  // 主键是 string 类型，GORM 不会自动生成
}
```

### 根本原因

1. `RemoteConfig.ID` 是 `string` 类型（与项目中其他模型如 `Snapshot.ID` 一致）
2. GORM 对 `string` 类型主键不会自动生成值（不像 `uint` 自增）
3. `AddRemote` 方法构建 `RemoteConfig` 时未设置 `ID` 字段
4. `Create` 方法直接使用传入的空 ID 插入数据库

### 对比其他模型

查看 `snapshot.go` 等模型，应使用 UUID 生成 ID。项目已依赖 `google/uuid` 包。

## 修复方案

在 `AddRemote` 方法中，构建 `RemoteConfig` 之前生成 UUID 作为 ID。

### 修改文件

1. **remote_method.go** — 在构建 config 前生成 UUID

### 具体修改

```go
import "github.com/google/uuid"

config := &models.RemoteConfig{
    ID:        uuid.New().String(),  // 新增：生成 UUID
    Name:      name,
    URL:       url,
    // ...
}
```

### 额外检查

需要确认以下操作是否依赖 ID：
- `Update` — 按 `config.ID` 更新 → 修复后正常
- `Delete` — 按 ID 删除 → 修复后正常
- `SetDefault` — 按 ID 设置 → 修复后正常

### 数据修复

已存在的无 ID 记录需要手动修复或删除后重新添加。

### 验收标准

1. `remote add <url>` 后数据库记录有正确的 UUID
2. `remote list` 正常显示
3. `remote remove <name>` 正常工作
4. 现有功能不受影响
5. `go test ./...` 全部通过
