# 表格行间分隔线 + scan rules 全表格化设计

## 背景

当前 `output.Table` 渲染的数据行之间没有水平分隔线，视觉上多行数据难以区分。
`scan rules` 命令输出仍使用纯文本缩进列表，未使用统一的表格渲染器。

## 修改 1：Table 行间分隔线

**文件**: `internal/cli/output/table.go`

在 `Render()` 方法中，数据行之间插入 `├──┼──┤` 水平分隔线。

改动前：
```
┌──────┬──────┐
│ Col1 │ Col2 │
├──────┼──────┤
│ v1   │ v2   │
│ v3   │ v4   │
└──────┴──────┘
```

改动后：
```
┌──────┬──────┐
│ Col1 │ Col2 │
├──────┼──────┤
│ v1   │ v2   │
├──────┼──────┤
│ v3   │ v4   │
└──────┴──────┘
```

## 修改 2：scan rules 全表格输出

**文件**: `internal/cli/cobra/root.go`（`printScanRuleList` 函数）

4 个分组全部使用 `output.Table` 渲染：

| 分组 | 列 | 空数据 |
|------|----|--------|
| 默认全局规则 | 路径 | — |
| 项目扫描模板 | 路径 | 不显示该分组 |
| 自定义规则 | 路径 | 显示"暂无" |
| 已注册项目 | 名称、路径 | 显示"暂无" |

每组前加粗标题，表格间空行分隔。

## 影响范围

- 所有使用 `Table` 的命令自动获得行间分隔线
- `scan rules` 输出格式变更
- 现有测试需要更新
