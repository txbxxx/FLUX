# 符号链接目录收集修复 PRD

## 背景

用户本地 `~/.claude/skills/` 下有多个 `lark-*` 符号链接子目录。执行 `snapshot create` 后，skills 目录完全没有出现在快照中（快照只有 settings.json 和 plugins）。

根因：`collector.go` 的 `collectFilesUnderDirWithDepth` 使用 `filepath.Walk` 遍历目录，Walk 默认不跟踪符号链接。代码尝试在检测到符号链接目录后递归调用自身，但递归入口仍然使用 `filepath.Walk`，导致套娃失败 — 符号链接目录内的文件始终无法被收集。

相关 Issue: txbxxx/FLUX#41

## 目标

1. 修复 `snapshot create` 无法收集符号链接目录内文件的问题
2. 快照文件保留用户视角路径（符号链接路径），不丢失上下文
3. 新增 `is_symlink` 和 `link_target` 字段，标识符号链接文件并记录真实路径
4. 保证存量数据库自动迁移（GORM AutoMigrate 加列）
5. sync push/pull 正确处理符号链接文件的序列化和反序列化

## 非目标（明确不做什么）

- 不实现恢复时自动重建符号链接（恢复统一写为普通文件）
- 不修改 detector 的扫描逻辑（detector 不涉及文件收集）
- 不处理 Windows 普通用户权限下无法创建符号链接的问题（那是 OS 层面的限制）

## 用户故事

1. 作为用户，我执行 `snapshot create -p claude-global` 时，希望 skills 下的符号链接子目录内的文件也能被正确收集进快照
2. 作为用户，我执行 `snapshot list` 后查看快照详情，希望能看到哪些文件来自符号链接以及它们的真实路径
3. 作为用户，我执行 `sync push` 后，希望远端仓库包含符号链接目录下的文件
4. 作为用户，我执行 `sync pull` 后，希望另一台机器也能拉取到符号链接目录下的文件

## 功能详情

### 1. 修复 collector 符号链接遍历

**文件**: `internal/service/snapshot/collector.go`

**方案**: 在 `collectFilesUnderDirWithDepth` 中，`filepath.Walk` 回调检测到符号链接目录时，使用 `os.ReadDir` 直接读取链接目标的子项列表，对每个子项构造完整路径后递归处理，而非再次调用 `collectFilesUnderDirWithDepth`（避免套娃）。

```
Walk 遍历 ~/.claude/
  → 遇到 skills/lark-im（符号链接目录）
  → os.Stat 确认目标是目录
  → filepath.EvalSymlinks 获取真实路径
  → os.ReadDir(真实路径) 逐项处理
  → 对每个子项递归 collectSingleFile 或 collectFilesUnderDirWithDepth
```

### 2. SnapshotFile 结构体新增字段

**文件**: `internal/models/snapshot.go`

```go
type SnapshotFile struct {
    // ... 现有字段不变
    IsSymlink  bool   `json:"is_symlink" gorm:"column:is_symlink;default:false"`            // 是否来自符号链接目录
    LinkTarget string `json:"link_target,omitempty" gorm:"column:link_target;type:TEXT"`     // 符号链接指向的真实路径
}
```

- `Path` 和 `OriginalPath` 保持符号链接视角的路径
- `IsSymlink` 标识文件是否来自符号链接目录
- `LinkTarget` 记录符号链接解析后的真实路径，仅在 `IsSymlink=true` 时有值

### 3. 数据库表变更

**文件**: `pkg/database/db.go`

- `snapshot_files` 表新增 `is_symlink` (INTEGER DEFAULT 0) 和 `link_target` (TEXT) 两列
- GORM `SnapshotFileRecord` 新增对应字段，AutoMigrate 自动加列
- 数据迁移逻辑（旧库→新库）同步更新 INSERT/SELECT 语句

### 4. sync push/pull 适配

**文件**: `internal/service/sync/packager.go` 等

- JSON 序列化/反序列化自动覆盖新字段（`json` tag 已定义）
- push 时打包新字段到远端
- pull 时从远端解析新字段写入本地数据库

## 技术方案

### 核心修复：collector.go 的 Walk 回调

```go
// 在 Walk 回调中，检测到符号链接目录时：
if !info.IsDir() && info.Mode()&os.ModeSymlink != 0 {
    realInfo, statErr := os.Stat(path)
    if statErr == nil && realInfo.IsDir() {
        // 解析真实路径
        realPath, _ := filepath.EvalSymlinks(path)

        // 循环防护：已在 seen 中则跳过
        if _, seen := seen[realPath]; seen {
            return nil
        }
        seen[realPath] = struct{}{}

        // 直接用 os.ReadDir 读取真实目录内容
        entries, err := os.ReadDir(realPath)
        if err != nil {
            errors = append(errors, CollectError{Path: path, Message: err.Error()})
            return nil
        }

        for _, entry := range entries {
            fullPath := filepath.Join(path, entry.Name()) // 保持符号链接视角的路径
            // 递归处理每个子项...
        }
        return nil
    }
}
```

关键点：`filepath.Join(path, entry.Name())` 使用原始符号链接路径拼接，保持用户视角。`LinkTarget` 则记录 `realPath`。
