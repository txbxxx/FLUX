# 快照恢复功能 PRD

> 创建时间：2026-04-11
> 状态：待开发
> 优先级：P0

---

## 1. 背景与目标

### 1.1 现状

用户可以创建、查看、删除配置快照，但无法恢复。快照只能"看"不能"用"，等于备份了却不能还原，功能闭环缺失。

### 1.2 目标

实现快照恢复功能，让用户能将历史快照中的配置文件写回到磁盘原始路径，完成"备份 → 恢复"的核心闭环。

### 1.3 非目标（后续阶段）

- 快照更新（snapshot update）
- Git 版本控制封装
- 远端仓库同步（push/pull）
- 历史版本选择（从 Git commit 恢复）
- TUI 恢复界面

---

## 2. 目标用户

AI 工具的重度使用者，满足以下场景：

- 使用多个 AI 工具（Claude、Codex 等），配置文件分散
- 定期创建快照备份配置
- 配置改坏后需要快速恢复到之前的正常状态
- 升级 AI 工具后发现问题，需要回退配置

---

## 3. 用户场景

### 场景 1：配置改坏了，一键回滚

> 小明修改 Claude 的 settings.json 后，AI 行为异常。他执行 `ai-sync snapshot restore "升级前备份"`，系统展示变更摘要，确认后配置恢复到正常状态。

### 场景 2：只想恢复某几个文件

> 小红升级 Codex 后只有 skills 目录出了问题。她执行 `ai-sync snapshot restore "升级前备份" --files skills/README.md,skills/config.json`，只恢复需要的文件，不影响其他配置。

### 场景 3：先预览再决定

> 小刚想恢复快照但不确定会改哪些文件。他执行 `ai-sync snapshot restore "初始配置" --dry-run`，看到变更摘要后决定是否恢复。

---

## 4. 功能需求

### FR-01 全量恢复

用户通过快照 ID 或名称恢复快照中的所有配置文件到原始路径。

- 支持通过 ID（完整或前缀匹配）或名称查找快照
- 恢复前自动备份当前磁盘上的配置文件
- 展示变更摘要（新增、更新、跳过）
- 需用户确认后才执行写入（`--force` 可跳过）

### FR-02 选择性恢复

用户指定要恢复的文件路径，只恢复部分文件。

- 通过 `--files` 参数指定文件路径，逗号分隔
- 指定的文件不在快照中时报错提示
- 未指定的文件不受影响

### FR-03 预览模式

用户在不写入磁盘的情况下预览恢复效果。

- `--dry-run` 触发预览模式
- 展示完整的变更摘要（与正式恢复一致）
- 不创建备份，不写入任何文件

### FR-04 强制备份

每次恢复（非 dry-run）自动备份当前配置。

- 备份目录：`~/.ai-sync/backup/<timestamp>/`
- 只备份快照中存在且磁盘上也存在的文件
- 备份路径在输出中展示，用户可手动访问
- 备份失败则中止恢复，不写入任何文件

---

## 5. 非功能需求

### NFR-01 安全性

- 恢复前必须备份（不可关闭）
- 写入前必须用户确认（`--force` 除外）
- 单个文件写入失败不影响其他文件

### NFR-02 性能

- 恢复 100 个文件（总大小 < 5MB）在 3 秒内完成
- 预览模式响应时间 < 1 秒

### NFR-03 兼容性

- 支持 Windows、macOS、Linux
- 兼容现有快照数据（不需要迁移）
- 不改变现有数据库表结构

---

## 6. 命令接口

```
ai-sync snapshot restore <id-or-name>                      # 全量恢复
ai-sync snapshot restore <id-or-name> --files <path1,path2> # 选择性恢复
ai-sync snapshot restore <id-or-name> --dry-run             # 仅预览
ai-sync snapshot restore <id-or-name> --force               # 跳过确认
```

## 7. 交互流程

```
用户执行 restore 命令
       ↓
  找到快照？──否──→ 报错："快照 'xxx' 不存在"
       ↓ 是
  指定了 --files？──是──→ 文件在快照中？──否──→ 报错："文件 'xxx' 不在快照中"
       ↓ 否                        ↓ 是
       ↓←←←←←←←←←←←←←←←←←←←←←←←←←←←
       ↓
  dry-run？──是──→ 展示摘要，结束
       ↓ 否
  创建备份 ──失败──→ 报错退出
       ↓ 成功
  展示变更摘要
       ↓
  force？──是──→ 直接恢复
       ↓ 否
  用户确认？──否──→ 提示"已取消"，备份保留
       ↓ 是
  写入文件
       ↓
  展示结果 + 备份路径
```

## 8. 输出规范

### 正常恢复输出

```
快照: 升级前备份 (550e8400-...)
即将恢复 12 个文件:
  更新: settings.json
  更新: .claude/CLAUDE.md
  新增: plugins/config.json
  跳过: skills/README.md (内容相同)

备份已创建: C:\Users\xxx\.ai-sync\backup\20260411-143022\

确认恢复？[y/N]: y

恢复完成！
  成功: 3 个文件
  跳过: 9 个文件
  失败: 0 个文件
  备份: C:\Users\xxx\.ai-sync\backup\20260411-143022\
```

### 无需恢复

```
快照: 升级前备份 (550e8400-...)
所有文件内容相同，无需恢复。
```

## 9. 验收标准

| 编号 | 验收条件 | 验证方法 |
|------|---------|---------|
| AC-01 | 全量恢复能将快照文件写回原始路径 | 创建快照 → 修改配置 → restore → 验证文件内容 |
| AC-02 | 选择性恢复只影响指定文件 | restore --files 指定文件 → 验证其他文件未变 |
| AC-03 | 恢复前自动创建备份 | restore 后检查备份目录存在且内容为恢复前的文件 |
| AC-04 | dry-run 不写入不备份 | dry-run 后检查文件和备份目录均未变化 |
| AC-05 | 快照不存在时报错 | restore 不存在的 ID → 看到错误提示 |
| AC-06 | --files 指定不存在的文件时报错 | restore --files 不存在的路径 → 看到错误提示 |
| AC-07 | 内容相同的文件被跳过 | 快照和磁盘内容一致 → 显示"无需恢复"或全部跳过 |
| AC-08 | 备份失败时中止恢复 | 模拟备份目录无权限 → 看到错误，文件未被修改 |
| AC-09 | 用户拒绝确认时安全退出 | 输入 N → 提示已取消，文件未被修改 |
| AC-10 | 单文件写入失败不影响其他文件 | 模拟某个文件无写权限 → 其他文件正常恢复 |
