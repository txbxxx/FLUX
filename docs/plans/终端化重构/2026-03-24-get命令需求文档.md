# AI Sync Manager `get` 命令需求文档

## 文档信息

- 日期：2026-03-24
- 范围：为终端版 AI Sync Manager 增加配置浏览与配置编辑能力
- 目标：补充 `get <app> <path>` 命令，让用户可以查看配置目录、查看配置文件内容，并在需要时进入终端编辑模式直接修改本机原文件

## 1. 背景

当前 `ai-sync scan` 只展示工具级摘要，例如：

- 找到了 `Codex` / `Claude`
- 根配置目录在哪里
- 顶层有哪些可同步项

但用户仍然缺少两个关键能力：

1. 无法继续向下查看配置目录内部到底有哪些文件
2. 无法直接查看某个配置文件的具体内容，更无法在终端内直接修改

这会导致当前命令只能回答“能同步什么”，不能回答：

- `skills\` 目录里到底有哪些技能文件
- 某个 `README.md`、`settings.json`、`rules` 文件具体写了什么
- 我能否直接在当前工具里修改这些真实配置

因此，需要补一个新的命令入口，而不是继续把 `scan` 做成递归巨量输出。

## 2. 设计目标

新增一个明确的终端命令：

```text
ai-sync get <app> <path>
```

它应支持两种基本能力：

1. 当目标是目录时，列出该目录下一层的配置项
2. 当目标是文件时，输出该文件内容

同时支持一个增强模式：

```text
ai-sync get <app> <path> --edit
```

其含义是：

- 进入类 `vim` 的终端编辑界面
- 显示行号
- 支持滚动
- 保存后直接修改本机原文件

## 3. 非目标

本次不做以下内容：

- 不把 `scan` 改成递归树状输出
- 不做任意路径读取器，不允许读取工具配置根目录之外的内容
- 不做多文件标签页编辑
- 不做语法高亮
- 不做撤销历史持久化
- 不做目录树交互浏览器
- 不做二进制文件编辑
- 不做项目级配置目录和全局配置目录之间的混合浏览

本次范围严格聚焦于：

- 指定一个工具
- 指定该工具配置根目录下的一个相对路径
- 看目录 / 看文件 / 编辑文件

## 4. 用户故事

### 4.1 查看目录

作为用户，我希望执行：

```text
ai-sync get codex skills\
```

然后看到 `skills\` 下当前一层有哪些目录和文件。

### 4.2 逐层向下查看

作为用户，我希望执行：

```text
ai-sync get codex skills\aiskills\
```

然后继续看到 `skills\aiskills\` 这一层的内容，而不是一次性把整个目录树全部展开。

### 4.3 查看文件内容

作为用户，我希望执行：

```text
ai-sync get codex skills\aiskills\README.md
```

然后直接在终端输出这个文件的完整内容。

### 4.4 编辑文件内容

作为用户，我希望执行：

```text
ai-sync get codex skills\aiskills\README.md --edit
```

然后进入全屏终端编辑界面，带行号、可滚动、可保存，退出后真实修改本机文件。

## 5. 命令语义

### 5.1 基本语法

```text
ai-sync get <app> <path>
ai-sync get <app> <path> --edit
```

其中：

- `<app>` 目前只支持 `codex`、`claude`
- `<path>` 为相对于该工具配置根目录的相对路径

### 5.2 工具根目录

命令内部应自动映射：

- `codex` -> 用户主目录下 `.codex`
- `claude` -> 用户主目录下 `.claude`

示例：

- `get codex skills\`
  实际针对 `C:\Users\<user>\.codex\skills`
- `get claude commands\`
  实际针对 `C:\Users\<user>\.claude\commands`

### 5.3 文件与目录判断规则

命令不能靠扩展名猜测文件类型，而应基于真实文件系统判断：

- 路径存在且 `isDir=true` -> 按目录处理
- 路径存在且 `isDir=false` -> 按文件处理

因此以下两个命令的差异由真实路径决定，而不是由输入末尾是否带 `\` 决定：

```text
ai-sync get codex skills\
ai-sync get codex skills\README.md
```

### 5.4 目录模式

当目标是目录时：

- 只列出当前一层子项
- 默认不递归
- 如果用户继续输入更深一级路径，再继续列那一层

例如：

```text
ai-sync get codex skills\
ai-sync get codex skills\aiskills\
ai-sync get codex skills\aiskills\prompts\
```

应分别展示不同层级的内容。

### 5.5 文件模式

当目标是文件且未指定 `--edit` 时：

- 直接输出文件完整内容到 stdout
- 不进入 TUI
- 不修改文件

### 5.6 编辑模式

当目标是文件且指定 `--edit` 时：

- 进入全屏终端编辑界面
- 显示行号
- 支持滚动和光标移动
- 保存后直接覆盖本机原文件
- 退出后回到普通终端

## 6. 输出要求

### 6.1 目录输出

目录模式输出必须满足：

- 人类可读
- 明确告诉用户当前查看的是哪个工具、哪个相对路径
- 能分辨目录与文件
- 路径显示使用相对工具根目录的相对路径

建议结构：

```text
Codex > skills\aiskills\

类型    名称                 路径
目录    prompts              skills\aiskills\prompts\
文件    README.md            skills\aiskills\README.md
文件    workflow.md          skills\aiskills\workflow.md
```

### 6.2 文件输出

文件模式输出必须满足：

- 直接输出原始文件内容
- 不额外包裹复杂样式
- 可以在输出前给出简短头部，告诉用户当前文件路径

建议结构：

```text
Codex > skills\aiskills\README.md

<文件正文原样输出>
```

### 6.3 编辑模式输出

编辑模式不走普通 stdout 文本输出，而是进入全屏终端编辑界面。

界面至少应包含：

- 文件标题
- 当前相对路径
- 行号
- 编辑区
- 快捷键提示
- 保存结果提示

## 7. 安全与边界要求

这是本次功能的硬约束。

### 7.1 不允许越界访问

`get` 只能访问目标工具配置根目录内、且属于允许同步范围内的文件和目录。

必须拒绝类似输入：

```text
ai-sync get codex ..\Desktop\a.txt
ai-sync get codex ..\..\Windows\System32
```

### 7.2 不允许把 `get` 变成任意文件浏览器

只有当前工具已定义的配置范围允许访问。

例如 `codex` 只能访问类似：

- `config.toml`
- `skills\...`
- `rules\...`
- `superpowers\...`

`claude` 只能访问类似：

- `settings.json`
- `CLAUDE.md`
- `commands\...`
- `plugins\...`
- `output-styles\...`

### 7.3 只编辑文本文件

如果目标是二进制文件：

- 普通模式不输出乱码正文
- 编辑模式直接拒绝并说明原因

## 8. 错误提示要求

错误输出必须是中文，并且说明原因。

至少覆盖以下场景：

- 工具名不支持
- 工具根目录不存在
- 请求路径不存在
- 请求路径超出允许范围
- 目标是目录却指定了不支持的行为
- 目标是二进制文件却尝试输出或编辑
- 保存文件失败

错误示例方向：

```text
读取失败：未找到路径 skills\missing.md
请先确认该配置文件是否存在
```

## 9. 快捷键需求

编辑模式至少需要以下快捷键：

- `Ctrl+S`：保存
- `Esc`：退出编辑器
- `Ctrl+C`：强制退出编辑器
- 方向键：移动光标
- `PgUp/PgDn`：滚动

如果实现成本允许，可补充：

- `Home/End`
- `Ctrl+F` / `Ctrl+B`
- `Ctrl+G` 显示当前位置

## 10. 与现有命令的关系

### 10.1 `scan`

`scan` 继续承担“摘要发现”职责：

- 看有哪些工具
- 看配置根目录
- 看顶层可同步项

### 10.2 `get`

`get` 承担“进入具体配置内容”职责：

- 看目录
- 看文件
- 编辑文件

二者是互补关系，不应混成一个命令。

## 11. 验收标准

完成后应满足：

- `ai-sync get codex skills\` 能列当前一层目录
- `ai-sync get codex skills\aiskills\` 能继续列下一层
- `ai-sync get codex skills\aiskills\README.md` 能输出文件内容
- `ai-sync get codex skills\aiskills\README.md --edit` 能进入全屏编辑模式
- 编辑器带行号
- 保存后真实修改本机原文件
- 不允许访问工具配置范围之外的路径
- 错误提示使用中文

## 12. 结论

`get` 命令的本质不是替代 `scan`，而是补足“配置内容可见、可读、可改”这一条关键链路。

最终命令职责应清晰分工：

- `scan`：告诉用户“有哪些配置”
- `get`：告诉用户“配置里面有什么”
- `get --edit`：让用户直接修改真实配置文件
