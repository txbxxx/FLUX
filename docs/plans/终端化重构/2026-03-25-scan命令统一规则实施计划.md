# Scan 命令统一规则 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 为终端版 AI Sync Manager 落地统一同步规则源，完成 `scan` 按 app 过滤、`scan add/remove/list`、关键文件门槛、Claude `plugins/` 文件级白名单，并统一 `scan`、`get`、`snapshot create` 的最终生效规则。

**Architecture:** 在现有 `runtime + usecase + Cobra + service` 结构上新增统一规则解析层。默认规则和关键文件定义保留在 `internal/service/tool`，用户自定义规则持久化到 SQLite；`scan` 只展示规则解析结果，`get` 与 `snapshot create` 不再各自维护白名单，而是统一依赖规则解析层。

**Tech Stack:** Go 1.25、Cobra、Bubble Tea、SQLite、Go 标准库 `os/filepath/time`

## 执行顺序说明

建议严格按以下顺序执行：

1. 先建表和 DAO
2. 再做规则定义与规则解析
3. 再做 usecase 与 CLI
4. 再接入 accessor 与 snapshot
5. 最后补 TUI 与文档

原因：

- `scan add/remove/list` 依赖规则存储
- `get` 与 `snapshot create` 依赖最终规则解析结果
- 如果先改 CLI 或 TUI，会反复返工结果结构
