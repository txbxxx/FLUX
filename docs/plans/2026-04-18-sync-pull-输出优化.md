# sync pull 输出优化 PRD

> 创建时间：2026-04-18
> 状态：待讨论
> 优先级：P1
> 前置依赖：sync pull 冲突交互改进（已合并）

---

## 1. 背景与目标

### 1.1 现状

sync pull 时自动解决（无冲突）的文件会逐个打印，导致以下问题：

1. **刷屏**：`market/`、`skill/` 等目录常有几十上百个 `.md` 文件，全部逐行打印会刷屏数十秒
2. **信息密度低**：用户对自动解决的文件无需逐条确认或审阅，看了也不会采取任何行动
3. **输出重复**：`sync_method.go:493-500`（UseCase 层）和 `sync.go:141-149`（CLI 层）对 `autoResolved` 文件各打印一遍，内容完全重复

### 1.2 目标

- 自动解决文件默认只显示**按目录聚合的统计摘要**，消除刷屏
- 需要看详情时可加 `--verbose` 展开完整文件列表
- 消除 UseCase 层和 CLI 层的重复输出
- 冲突文件列表的展示不受影响（已经是逐文件交互，有实际作用）

---

## 2. 非目标

- 不改变冲突文件的交互逻辑（逐文件选择是必要的）
- 不改变 sync push 的输出格式
- 不做自动解决文件的交互式确认（自动解决就是自动的，不需要确认）

---

## 3. 功能详情

### 3.1 自动解决文件按目录聚合展示

将逐行打印改为按目录聚合 + 计数：

**改前**（当前行为，大量文件时刷屏）：

```
自动同步（无冲突）：
  - market/aaa.md — 远端新增文件（320 字节），本地不存在此文件，已自动同步到本地快照
  - market/bbb.md — 远端新增文件（280 字节），本地不存在此文件，已自动同步到本地快照
  - market/ccc.md — 远端新增文件（410 字节），本地不存在此文件，已自动同步到本地快照
  ... (重复 40+ 行)
  - skill/xxx.md — 远端新增文件（150 字节），本地不存在此文件，已自动同步到本地快照
  - skill/yyy.md — 远端新增文件（200 字节），本地不存在此文件，已自动同步到本地快照
  ... (重复 20+ 行)
  - settings.json — 本地快照有文件，远端已无此文件，保留本地版本不变
```

**改后**（默认，简洁）：

```
自动同步（无冲突）：
  market/ — 45 个远端新增文件
  skill/ — 22 个远端新增文件
  settings.json — 本地独有文件，已保留
```

聚合规则：
- 同一目录下的文件按**类型**（远端新增 / 本地独有）计数合并
- 根目录下的文件（无目录前缀）仍逐个展示（数量少，信息密度高）
- 目录层级取第一级（`market/aaa.md` → `market/`，不取 `market/sub/`）
- 展示格式：`<目录>/ — <数量> 个<类型描述>`

### 3.2 `--verbose` flag 展开完整列表

`sync pull` 新增 `--verbose` / `-v` flag：

- **不加 flag**（默认）：按目录聚合展示
- **加 `--verbose`**：恢复当前的逐文件展示

```
$ fl sync pull -p claude-global --verbose
# 展示完整文件列表（与当前行为一致）
```

### 3.3 消除 UseCase 层重复输出

当前问题：
- `sync_method.go:493-500`（UseCase 层）用 `fmt.Println` 直接打印自动解决文件
- `sync.go:141-149`（CLI 层）的 `printSyncPullResult` 再次打印

改进方案：
- **UseCase 层不再直接打印自动解决文件列表**，只将 `autoResolved` 数据放入返回值 `SyncPullResult`
- **CLI 层统一负责展示**，在 `printSyncPullResult` 中根据 `--verbose` 决定展示粒度
- UseCase 层仅保留**冲突交互**的直接打印（因为交互式提示必须在 UseCase 层完成）

### 3.4 冲突文件列表展示不变

冲突文件列表已有 `showLimit := 10` 的截断和交互式确认，是必要信息，不做改动。

## 4. 技术方案

### 4.1 sync pull 新增 `--verbose` flag

**文件**：`internal/cli/cobra/sync.go`

```go
// syncPullCmd 新增 flag
syncPullCmd.Flags().BoolP("verbose", "v", false, "显示完整的自动同步文件列表")
```

在调用 UseCase 时将 verbose 参数传入。

### 4.2 新增聚合展示函数

**文件**：`internal/cli/cobra/sync.go`

```go
// printAutoResolvedSummary 按目录聚合展示自动解决的文件
func printAutoResolvedSummary(w io.Writer, autoResolved []typesSync.AutoResolvedInfo, verbose bool)
```

聚合逻辑：
1. 遍历 `autoResolved` 列表
2. 提取每个文件路径的第一级目录（如 `market/aaa.md` → `market/`）
3. 按目录 + 类型聚合计数
4. 根目录下的文件单独列出
5. `verbose=true` 时回退到逐文件展示

### 4.3 移除 UseCase 层的自动解决打印

**文件**：`internal/app/usecase/sync_method.go`

删除 `sync_method.go:493-500` 的自动解决打印代码：

```go
// 删除以下代码块
if len(autoResolved) > 0 {
    fmt.Println()
    fmt.Println("自动同步（无冲突）：")
    for _, ar := range autoResolved {
        fmt.Printf("  - %s — %s\n", ar.Path, ar.Summary)
    }
    fmt.Println()
}
```

同时确保 `autoResolved` 数据正确传入 `SyncPullResult.AutoResolved` 返回值，由 CLI 层统一展示。

### 4.4 改动文件清单

| 文件 | 改动类型 | 说明 |
|------|---------|------|
| `internal/cli/cobra/sync.go` | 修改 | 新增 `--verbose` flag、新增聚合展示函数、移除 CLI 层重复打印 |
| `internal/app/usecase/sync_method.go` | 修改 | 移除 UseCase 层自动解决文件的直接打印 |
| `internal/app/usecase/types.go` 或相关 | 修改 | `SyncPullInput` 新增 `Verbose` 字段（如需传入） |

## 6. 验收标准

| 测试场景 | 预期结果 |
|---------|---------|
| 默认 sync pull，自动解决 50 个文件 | 按目录聚合展示，不超 10 行 |
| `sync pull --verbose`，自动解决 50 个文件 | 逐文件展示完整列表（与当前行为一致） |
| 自动解决 0 个文件 | 不展示"自动同步"区块 |
| 自动解决 2 个文件（根目录级别） | 逐文件展示（不聚合，数量太少无需聚合） |
| 冲突文件交互 | 行为不变，逐文件选择 |
| UseCase 层不打印自动解决文件 | 自动解决信息只在最终结果中出现一次 |
| `go test ./...` | 全部通过 |
