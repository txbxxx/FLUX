# AI Sync Manager 首页重构设计草案

## 文档信息

- 日期：2026-03-20
- 范围：首页重构
- 状态：草案，等待确认
- 触发原因：当前首页虽然已经具备基础视觉和部分真实数据，但在首次使用场景下可理解性差、操作反馈弱、信息优先级错误，导致用户难以判断下一步该做什么

## 1. 当前问题总结

这次重构不是“调样式”，而是“重做首页的信息架构”。

根据当前实际使用反馈，首页存在以下根问题：

### 1.1 首屏任务不明确

用户进入页面后，虽然看到了“推荐下一步”，但并不能真正理解：

- 为什么先做扫描
- 扫描会得到什么
- 扫描之后下一步是什么
- 当前按钮是否真的生效

结果是首页看起来像有很多东西，但没有明确行动路径。

### 1.2 重要操作缺少即时反馈

当前首页最关键的按钮是“重新扫描”，但点击后没有明显反馈，容易让用户认为：

- 没有点击成功
- 系统没有响应
- 页面只是静态展示

对于首次使用者，这是非常致命的问题。

### 1.3 状态信息占比过高，但价值不高

目前首页把 `环境概览` 和 `流程概览` 放在较大区域展示，但对首次进入的用户来说：

- 环境概览并不能帮助他们完成第一步
- 流程概览过于抽象，不解决“现在我该怎么做”
- 这两块抢走了本该属于主操作区的注意力

### 1.4 首页混合了三种页面角色

当前首页同时承担：

- 首次使用引导页
- 仪表盘概览页
- 快速入口页

这是造成混乱的根本原因。  
首页必须只保留一个核心角色，否则用户永远不知道该先看哪里。

### 1.5 左侧导航过重

当前左侧导航：

- 宽度偏大
- 信息价值不高
- 视觉存在感过强

它在首页阶段抢占了主内容区空间，但并没有提供相应价值。

## 2. 重构目标

首页重构后的目标不是“更漂亮”，而是“更容易开始使用”。

需要同时满足这四个目标：

1. 用户进入后，5 秒内知道第一步该做什么
2. 每个核心按钮点击后都有明确反馈
3. 与当前动作无关的信息降级，不再抢占主视觉
4. 页面更像“开始向导”，而不是“半个后台首页”

## 3. 重构后的首页定位

首页应重新定位为：

> 面向首次使用和低频回访用户的“开始页 / 向导入口页”

它的职责应缩减为三件事：

1. 明确告诉用户第一步是什么
2. 提供第一步操作入口
3. 用最少的信息说明后续流程

首页不再承担：

- 大面积状态总览
- 并列展示多个同权重模块
- 类后台仪表盘的信息堆叠

## 4. 方案比较

### 方案 A：保留现有结构，继续缩减信息

做法：

- 保留当前大 Hero 区
- 缩小环境概览和流程概览
- 保留四张操作卡

优点：

- 改动成本低
- 可复用当前结构

缺点：

- 本质问题不会消失
- 仍然是“看起来很多，但不知道先做什么”

### 方案 B：重做为“单主任务首页”

做法：

- 首页只突出一个主任务：扫描本机配置
- 其他内容全部降级到辅助层
- 后续步骤改成折叠说明或紧凑步骤条

优点：

- 首屏任务极其清晰
- 首次使用更友好
- 页面噪音明显减少

缺点：

- 需要重做首页结构
- 当前的双栏概览区会被废弃

### 方案 C：改成 onboarding 弹窗流

做法：

- 首页极简
- 进入后直接用弹窗或引导面板一步一步带用户走

优点：

- 引导性最强

缺点：

- 过于“流程化”
- 不适合后续回访用户长期使用

### 推荐方案

推荐采用 `方案 B：单主任务首页`。

原因很直接：

- 当前最需要解决的是“用户不知道怎么开始”
- 不是“首页信息不够多”
- 也不是“视觉不够完整”

所以首页第一屏必须只保留一个主任务。

## 5. 新首页结构

首页重构后建议改成四层结构。

### 5.1 顶部：轻量页面头

只保留：

- 标题：`开始同步你的 AI 配置`
- 一句短说明：`先扫描本机配置，再创建快照并进入多设备同步`

不要在这一层塞状态胶囊和复杂按钮。

### 5.2 主操作区：单主任务卡

这是整个首页唯一的视觉焦点。

只放一个主卡片：

- 标题：`先扫描你的本机配置`
- 说明：解释会扫描什么、不会做什么
- 主按钮：`开始扫描`
- 次按钮：`查看支持范围`

主卡片下方直接显示扫描结果反馈：

- 未开始：`尚未扫描`
- 扫描中：`正在扫描本机工具与配置目录...`
- 成功：`已检测到 2 个工具，可继续创建快照`
- 失败：`扫描失败，请重试`

### 5.3 紧凑状态条：折叠前的简报层

原来的 `环境概览` 改成一行紧凑状态条，放在主卡片下方。

只保留 3 个值：

- 工具数
- 快照数
- 加密状态

展示方式建议：

- 横向三个小面板
- 信息短，不展开
- 只作为“补充判断”，不作为首页主内容

### 5.4 后续步骤区：折叠式步骤说明

原来的 `流程概览` 改成折叠说明，而不是大卡片。

建议做成：

- `1. 扫描本机配置`
- `2. 创建快照`
- `3. 连接仓库`
- `4. 在新设备恢复`

默认只展开当前步骤。  
用户如果不关心流程细节，可以完全忽略这一块。

### 5.5 最近记录区：降级展示

首页保留最近快照摘要，但权重继续下降。

建议：

- 只保留 `最近快照`
- `最近同步` 可以暂时隐藏或折叠
- 如果没有任何快照，就只显示一行空状态说明

原因：

在当前阶段，“最近同步”对首次使用价值很低。

## 6. 交互重构

### 6.1 扫描按钮

当前问题：

- 点了没有感知

重构后要求：

- 点击瞬间进入 loading
- 按钮文案变为 `扫描中...`
- 主卡状态同步变化
- 扫描成功后出现明确成功文案
- 扫描失败后出现错误文案和重试按钮

### 6.2 查看支持范围

当前问题：

- 没有反馈
- 占据主操作层但没有结果

重构后要求：

- 点击后弹出模态层
- 模态内容包括：
  - 当前支持的工具
  - 当前会扫描的全局目录
  - 当前不会处理的内容

这类说明适合弹窗，不适合占首页主空间。

### 6.3 创建快照入口

首页不再把创建快照做成首屏并列主卡。

扫描完成后，在主任务卡下方提供清晰后续动作：

- `下一步：创建第一份快照`

点击后再跳转到快照页。

## 7. 左侧导航调整

### 7.1 当前问题

- 导航太宽
- 对首次使用者帮助不大
- 占据横向空间

### 7.2 重构建议

左侧导航改成更窄的轻量版本。

建议：

- 宽度明显缩小
- 减少品牌区高度
- Logo 和标题压缩
- 导航项只保留图标 + 简短文字

如果需要进一步简化，可考虑：

- 首页状态下导航弱化
- 非首页时恢复正常对比度

## 8. 页面文案策略

新首页文案必须满足两个原则：

1. 解释当前动作
2. 解释动作结果

例如主卡片文案建议：

### 初始状态

- 标题：先扫描你的本机配置
- 描述：我们会识别 Codex、Claude 等本机配置目录，不会直接上传你的数据。
- 主按钮：开始扫描
- 次按钮：查看支持范围

### 扫描成功

- 状态：已检测到 2 个工具
- 下一步提示：现在可以创建第一份快照，固定当前环境基线。
- 行动按钮：前往创建快照

### 扫描失败

- 状态：扫描失败
- 辅助文案：请检查本地工具配置目录是否存在，然后重试。
- 行动按钮：重新扫描

## 9. 页面模块删改建议

### 应删除或降级的模块

- 大号 `推荐下一步`
- 大面积 `环境概览`
- 大面积 `流程概览`
- 首页并列四张同权重操作卡
- 首页中价值较低的 `最近同步`

### 应保留的模块

- 扫描入口
- 最近快照摘要
- 左侧导航
- 基础状态信息

### 应新增的模块

- 扫描成功/失败反馈
- 查看支持范围弹窗
- 折叠式步骤说明

## 11. 实施优先级建议

如果确认重构，建议按以下顺序执行：

1. 重做首页结构，移除双栏概览区
2. 给扫描按钮补完整反馈链路
3. 添加“查看支持范围”弹窗
4. 压缩左侧导航宽度
5. 将后续步骤改成折叠式说明
6. 最后再精修视觉

## 12. 结论

当前首页的主要失败点不是视觉细节，而是：

- 信息太多
- 主任务不够单一
- 交互反馈不够明确
- 导航和概览抢占主空间

因此，首页应该从“工具控制台首页”重构为“单主任务开始页”。

这是一次方向性调整，而不是继续堆 UI 模块。

## 13. Chrome MCP 真实页面评审结论

本轮已经在真实运行中的 `wails dev` 页面里，分别查看了：

- `当前版`
- `单主任务版`
- `轻引导版`

三者都使用同一套真实首页数据，并验证了：

- 可以在开发环境下切换
- 切换后内容即时变化
- 刷新页面后能保留最后选择
- “查看支持范围”弹层可正常打开

### 13.1 当前版

优点：

- 信息完整
- 已经具备真实数据和后续入口
- 更像成熟的工具型首页

缺点：

- 仍然太像“控制台首页”
- 首屏信息量偏大
- `环境概览` 与 `流程概览` 仍然占用较多注意力
- 对首次使用者来说，“为什么先做扫描”还是不够直接

结论：

- 可以作为“回访用户仪表盘”方向继续保留
- 但不适合作为当前阶段的最终首页

### 13.2 单主任务版

优点：

- 首页目标最清晰
- 用户一进入就能知道第一步是扫描
- 扫描结果、下一步动作、简报状态都围绕同一任务展开
- 后续步骤被成功降级，不再抢主焦点
- 与用户反馈里提到的“乱、看不懂、不会用”问题最匹配

缺点：

- 侧栏仍然偏宽
- 最近快照区域还可以继续收紧
- 文案和状态条还可以再进一步精炼

结论：

- 这是当前最适合作为最终首页方向的版本
- 后续应该在这一版上继续做正式重构，而不是回到当前版继续堆信息

### 13.3 轻引导版

优点：

- 文案解释更直接
- 更适合首次打开时建立认知
- 页面更轻，阅读压力更小

缺点：

- 更像 onboarding 模块，而不是长期使用的产品首页
- 操作密度偏低
- 对回访用户来说效率不如 `单主任务版`

结论：

- 适合作为首次使用引导语言参考
- 不适合作为最终首页结构

## 14. 最终推荐

最终推荐保留：

- `单主任务版` 作为首页正式方向

并吸收 `轻引导版` 的一部分优点：

- 保留更直接的说明文案
- 保留“为什么先扫描”的解释方式

不建议继续保留 `当前版` 作为正式首页主结构。  
它可以保留为未来“概览页 / 仪表盘页”的参考，但不适合继续承担首次进入时的开始页角色。
