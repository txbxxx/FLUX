# AI Sync Manager 测试规范

## 1. 测试分层

### 1.1 单元测试（Unit Tests）

**定义**：测试单个函数、方法或类的行为，不依赖外部系统。

**适用场景**：
- Models 层的数据结构验证
- Service 层的业务逻辑
- Utility 函数
- 配置解析

**示例位置**：
```
internal/models/models_test.go
internal/service/tool/paths_test.go
internal/service/crypto/service_test.go
```

**特点**：
- 执行速度快（毫秒级）
- 无外部依赖
- 可并行执行
- 可mock所有依赖

---

### 1.2 集成测试（Integration Tests）

**定义**：测试多个模块之间的交互，可能涉及文件系统、数据库等。

**适用场景**：
- UseCase 层的流程编排
- Service 层与 DAO 的交互
- 文件扫描和读取
- 配置解析器

**示例位置**：
```
internal/app/usecase/local_workflow_test.go
internal/app/usecase/config_browser_test.go
internal/service/tool/detector_test.go
internal/service/snapshot/snapshot_test.go
```

**特点**：
- 执行速度中等（秒级）
- 可能涉及文件系统
- 可能涉及内存数据库
- 需要清理临时资源

---

### 1.3 E2E测试（End-to-End Tests）

**定义**：测试完整的用户操作流程，模拟真实使用场景。

**适用场景**：
- CLI 命令的完整执行流程
- TUI 的用户交互流程
- 跨多个 UseCase 的复杂业务流程

**示例位置**：
```
internal/cli/cobra/root_test.go
cmd/fl/main_test.go
```

**特点**：
- 执行速度较慢（可能需要数秒）
- 涉及所有层级
- 尽量较少，覆盖关键路径

---

## 2. 测试命名规范

### 2.1 测试文件命名

**规则**：测试文件与被测文件同目录，命名为 `原文件名_test.go`

**示例**：
```
internal/models/snapshot.go           → internal/models/snapshot_test.go
internal/service/crypto/service.go    → internal/service/crypto/service_test.go
internal/cli/cobra/root.go           → internal/cli/cobra/root_test.go
```

### 2.2 测试函数命名

**单元测试**：
```go
func Test<函数名>(t *testing.T)
```

**示例**：
```go
func TestNewSnapshot(t *testing.T)
func TestValidateSnapshot(t *testing.T)
func TestGetFileExtension(t *testing.T)
```

**子测试**：
```go
func Test<函数名>_<测试场景>(t *testing.T)
```

**示例**：
```go
func TestDetectWithOptions_EmptyResult(t *testing.T)
func TestDetectWithOptions_CodexGlobal(t *testing.T)
func TestDetectWithOptions_ProjectScope(t *testing.T)
```

**基准测试**：
```go
func Benchmark<函数名>(b *testing.B)
```

**示例**：
```go
func BenchmarkDetectWithOptions(b *testing.B)
func BenchmarkScan(b *testing.B)
```

## 3. 测试组织结构

### 3.1 表格驱动测试（Table-Driven Tests）

**定义**：使用测试用例表格，一次编写多个测试场景。

**适用场景**：
- 验证函数的多种输入输出
- 测试边界条件
- 测试错误处理

**示例**：
```go
func TestValidateSnapshot(t *testing.T) {
    tests := []struct {
        name     string
        snapshot *Snapshot
        wantErr  bool
    }{
        {
            name: "有效快照",
            snapshot: &Snapshot{
                Message: "Valid snapshot",
                Project: "codex-global",
            },
            wantErr: false,
        },
        {
            name:     "空快照",
            snapshot: nil,
            wantErr:  true,
        },
        {
            name: "空消息",
            snapshot: &Snapshot{
                Project: "codex-global",
            },
            wantErr: true,
        },
    }

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            err := Validate
Snapshot(tt.snapshot)
            if tt.wantErr {
                assert.Error(t, err)
            } else {
                assert.NoError(t, err)
            }
        })
    }
}
```

---

### 3.2 测试辅助函数

**定义**：用于创建测试环境、Mock 对象、清理资源的辅助函数。

**命名规范**：
```go
func Create<Mock对象名称>(t *testing.T, <参数>...) <返回类型>
func Cleanup<Mock对象名称>(t *testing.T)
func Get<状态>ForTest() <返回类型>
```

**示例**：
```go
// 创建模拟项目
func CreateMockProject(t *testing.T, name string, hasConfig bool, deepNested bool) string

// 清理所有模拟项目
func CleanupMockProjects(t *testing.T)

// 创建模拟配置目录
func CreateMockConfigDir(t *testing.T, toolType ToolType, scope ConfigScope) string

// 清理模拟目录
func CleanupMockDir(t *testing.T, dir string)

// 获取测试用用户主目录
func GetUserHomeDirForTest() string
```

**实现原则**：
1. 在临时目录创建资源
2. 路踪创建的资源以便清理
3. 使用 `t.Cleanup()` 注册清理函数
4. 确保清理不返回错误

## 4. Mock 和测试数据

### 4.1 文件系统 Mock

**原则**：
- 使用 `t.TempDir()` 创建临时目录
- 创建完成后立即注册清理
- 不依赖真实用户配置目录

**示例**：
```go
func TestFileAccess(t *testing.T) {
    // 创建临时目录
    tempDir := filepath.Join(os.TempDir(), "test-project")
    _ = os.MkdirAll(tempDir, 0755)
    defer os.RemoveAll(tempDir)

    // 创建测试文件
    testFile := filepath.Join(tempDir, "config.toml")
    _ = os.WriteFile(testFile, []byte("key = \"value\""), 0644)

    // 执行测试
    result, err := ReadConfig(testFile)
    assert.NoError(t, err)
    assert.Equal(t, "value", result.Key)
}
```

---

### 4.2 数据库 Mock

**原则**：
- 使用内存数据库（SQLite 的 `:memory:` 或临时文件）
- 每个测试使用独立的数据库实例
- 测试完成后自动清理

**示例**：
```go
func TestSnapshotCRUD(t *testing.T) {
    // 创建临时数据库
    tempDB := filepath.Join(t.TempDir(), "test.db")
    db, err := database.New(tempDB)
    assert.NoError(t, err)
    defer db.Close()

    // 创建 DAO
    dao := models.NewSnapshotDAO(db)

    // 执行测试
    snapshot := models.NewSnapshot("Test message", "codex-global")
    err = dao.Create(snapshot)
    assert.NoError(t, err)
    assert.NotEmpty(t, snapshot.ID)
}
```

**位置**：`internal/tests/` 目录（或各包下的 `testdata/` 子目录）

## 5. 测试断言和验证

### 5.1 使用 testify/assert

**常用断言**：
```go
import "github.com/stretchr/testify/assert"

// 相等断言
assert.Equal(t, expected, actual)
assert.NotEqual(t, notExpected, actual)

// 布断言
assert.Nil(t, actual)
assert.NotNil(t, actual)
assert.Empty(t, collection)
assert.NotEmpty(t, collection)

// 布尔断言
assert.True(t, condition)
assert.False(t, condition)

// 比较断言
assert.Greater(t, actual, min)
assert.Less(t, actual, max)
assert.Contains(t, collection, element)

// 错误断言
assert.NoError(t, err)
assert.Error(t, err)
assert.ErrorIs(t, err, expectedError)
```

---

### 5.2 类型断言

**示例**：
```go
import "github.com/stretchr/testify/assert"

// 类型断言
assert.Implements(t, actual, (*MyInterface)(nil))
assert.IsType(t, (*MyStruct)(nil), actual)

// 数组/切片断言
assert.Len(t, slice, expectedLength)
assert.ElementsMatch(t, expectedSlice, actualSlice)
```

---

### 5.3 错误消息断言

**示例**：

## 8. CI/CD 集成

### 8.1 GitHub Actions 配置示例

```yaml
name: Tests

on:
  push:
    branches: [ master, main ]
  pull_request:
    branches: [ master, main ]

jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        go-version: ['1.21', '1.22']

    steps:
    - uses: actions/checkout@v4

    - name: Set up Go
      uses: actions/setup-go@v5
      with:
        go-version: ${{ matrix.go-version }}

    - name: Install dependencies
      run: go mod download

    - name: Run tests
      run: go test -v -race -coverprofile=coverage.out ./...

    - name: Check coverage
      run: |
        go tool cover -func=coverage.out.out
        COVERAGE=$(go tool cover -func=coverage.out.out | grep total | awk '{print $3}' | sed 's/%//')
        echo "Coverage: $COVERAGE%"
        if (( $(echo "$COVERAGE < 80" | bc -l) )); then
          echo "Coverage is below 80%"
          exit 1
        fi
```

## 9. 测试最佳实践

### 9.1 AAA 模式

**Arrange（准备）**：设置测试前置条件
**Act（执行）**：执行被测操作
**Assert（断言）**：验证结果

**示例**：
```go
func TestAddSnapshot(t *testing.T) {
    // Arrange
    tempDB := setupTestDatabase(t)
    dao := models.NewSnapshotDAO(tempDB)
    snapshot := models.NewSnapshot("Test message", "codex-global")

    // Act
    err := dao.Create(snapshot)

    // Assert
    assert.NoError(t, err)
    assert.NotEmpty(t, snapshot.ID)
}
```

---

### 9.2 测试独立性

**原则**：
- 每个测试应该独立运行
- 测试之间不应共享状态
- 使用 `t.TempDir()` 创建独立工作目录
- 每个测试使用独立的数据库实例

---

### 9.3 测试可读性

**原则**：
- 测试名称应清晰描述测试内容
- 使用子测试区分不同场景
- 测试失败时能快速定位问题

**示例**：
```go
// 好的命名
func TestSnapshotCreation_Succeeds(t *testing.T)
func TestSnapshotCreation_FailsWithEmptyMessage(t *testing.T)
func TestSnapshotCreation_FailsWithEmptyProject(t *testing.T)

// 不好的命名
func TestSnapshot1(t *testing.T)
func TestSnapshot2(t *testing.T)
func TestSnapshot3(t *testing.T)
```

---

### 9.4 测试性能

**原则**：
- 单元测试应在毫秒级完成
- 集成测试应在秒级完成
- 对关键路径使用基准测试

**示例**：
```go
// 单元测试（快速）
func TestGetFileExtension(t *testing.T) {
    result := GetFileExtension("config.toml")
    assert.Equal(t, "toml", result)
}

// 集成测试（中等速度）
func TestDetectWithOptions(t *testing.T) {
    // 设置临时目录、数据库
    // 执行扫描（可能需要1-2秒）
}

// 基准测试（测量性能）
func BenchmarkDetectWithOptions(b *testing.B) {
    for i := 0; i < b.N; i++ {
        detector.DetectWithOptions(ctx, opts)
    }
}
```

---

### 9.5 测试清理

**原则**：
- 使用 `defer` 确保资源清理
- 使用 `t.Cleanup()` 注册清理函数
- 清理不返回错误（或忽略错误）

**示例**：
```go
func TestWithResources(t *testing.T) {
    // 方法 1：使用 defer
    tempDir := os.TempDir()
    defer os.RemoveAll(tempDir)

    // 方法 2：使用 t.Cleanup
    t.Cleanup(func() {
        os.RemoveAll(tempDir)
    })
}
```

## 11. 常见测试问题

### 11.1 测试依赖真实系统

**问题**：测试依赖用户主目录、网络等外部系统

**解决方案**：
- 使用 `t.TempDir()` 创建临时目录
- Mock 外部依赖
- 设置测试专用环境变量

---

### 11.2 测试之间共享状态

**问题**：测试失败后影响其他测试

**解决方案**：
- 每个测试使用独立的实例
- 不使用全局变量
- 使用临时文件/数据库

---

### 11.3 清理失败导致资源泄漏

**问题**：测试失败后临时资源未清理

**解决方案**：
- 使用 `defer` 确保清理
- 使用 `t.Cleanup()` 注册清理函数
- 测试清理函数不返回错误

---

### 11.4 测试执行过慢

**问题**：测试套件执行时间过长

**解决方案**：
- 分离慢速测试到单独文件
- 使用并行测试
- 减少 E2E 测试数量
- 使用 Mock 替代真实操作

## 13. 示例：完整的测试文件

### 13.1 Models 层测试示例

```go
package models

import (
    "testing"
    "time"

    "github.com/stretchr/testify/assert"
)

func TestNewSnapshot(t *testing.T) {
    // Arrange
    message := "Test snapshot"
    projectName := "codex-global"

    // Act
    snapshot := NewSnapshot(message, projectName)

    // Assert
    assert.NotNil(t, snapshot)
    assert.NotEmpty(t, snapshot.ID)
    assert.Equal(t, message, snapshot.Message)
    assert.Equal(t, projectName, snapshot.Project)
    assert.False(t, snapshot.CreatedAt.IsZero())
}

func TestValidateSnapshot(t *testing.T) {
    tests := []struct {
        name     string
        snapshot *Snapshot
        wantErr  bool
    }{
        {
            name: "有效快照",
            snapshot: &Snapshot{
                Message: "Valid snapshot",
                Project: "codex-global",
            },
            wantErr: false,
        },
        {
            name:     "空快照",
            snapshot: nil,
            wantErr:  true,
        },
    }

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            err := ValidateSnapshot(tt.snapshot)
            if tt.wantErr {
                assert.Error(t, err)
            } else {
                assert.NoError(t, err)
            }
        })
    }
}
```
