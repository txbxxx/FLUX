### 3. 服务内部类型

```go
// CollectOptions 文件收集选项
type CollectOptions struct {
    Tools      []string               // 要收集的工具
    ProjectPath string                 // 项目路径
    Categories  []models.FileCategory  // 要包含的文件类别
    Excludes    []string               // 排除的文件模式

// CollectResult 文件收集结果
type CollectResult struct {
    Files    []models.SnapshotFile  // 收集的文件列表
    TotalSize int64                  // 总大小
    Errors   []CollectError          // 收集过程中的错误

// FileChangeStatus 文件变更状态
const (
    FileStatusCreated  FileChangeStatus = "created"   // 新建
    FileStatusUpdated  FileChangeStatus = "updated"   // 更新
    FileStatusDeleted  FileChangeStatus = "deleted"   // 删除
    FileStatusSame     FileChangeStatus = "same"      // 无变化
    FileStatusConflict FileChangeStatus = "conflict"  // 冲突
)
```

## 技术亮点

### 1. 文件哈希计算
```go
func (c *Collector) calculateHash(content []byte) string {
    hash := sha256.Sum256(content)
    return hex.EncodeToString(hash[:])
}
```

### 2. 二进制文件检测
```go
func (c *Collector) isBinaryFile(content []byte) bool {
    limit := 512
    if len(content) < limit {
        limit = len(content)
    }
    for i := 0; i < limit; i++ {
        if content[i] == 0 {
            return true  // NULL 字节表示二进制
        }
    }
    return false
}
```

## 数据流

```
创建快照:
用户输入 → Service.CreateSnapshot()
         → Collector.Collect()
         → 收集文件内容、计算哈希、分类
         → SnapshotDAO.Create()
         → 保存到数据库

应用快照:
用户选择快照 → Applier.ApplySnapshot()
            → 检测冲突、创建备份（可选）
            → 遍历文件并写入
            → 返回应用结果

比较快照:
用户选择两个快照 → Comparator.CompareSnapshots()
                → 逐文件比较哈希
                → 返回变更摘要
```

## 已知限制

### 1. 数据库文件内容存储
- **现状**: 快照文件内容存储在内存中，没有持久化到 snapshot_files 表
- **影响**: 通过 DAO 读取的快照不包含 Files 数据
- **解决方案**:
  - 短期：使用内存中的快照对象
  - 长期：扩展 DAO 支持文件表联合查询

- **现状**: 31.8% 覆盖率
- **原因**: 部分功能需要真实文件系统环境
